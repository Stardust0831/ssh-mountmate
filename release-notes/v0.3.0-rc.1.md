# SSH MountMate v0.3.0-rc.1

Pre-release introducing truthful upload visibility, verified remote refresh, Explorer commands, and dual packaging.

## Changes

- Read each mount's rclone `vfs/queue`, `vfs/stats`, and `core/stats` endpoints to show queued files and real remote-transfer progress.
- Add a transfer center that can open automatically when uploads start without taking focus.
- Warn before unmounting or exiting while files remain queued or uploading.
- Upgrade refresh to forget cached entries, actively refresh the VFS directory, and verify the result with a direct remote listing.
- Add connection-card context menus for Open, Refresh, Transfers, and Logs.
- Add Settings actions and same-executable CLI modes to register or remove Windows Explorer Refresh and Transfers commands.
- Forward Explorer and second-launch commands to the running GUI over authenticated loopback IPC.
- Build and publish both backward-compatible onefile packages and faster-starting `-onedir.zip` packages for every platform and architecture.

## Verification

- `python3 -m compileall -q src tests build launcher.py`
- `PYTHONPATH=src python3 -m unittest discover -s tests -v`
- `git diff --check`

# SSH MountMate v0.3.0-rc.1 中文说明

这个预发布版新增真实上传可见性、远端刷新核验、资源管理器命令和双构建格式。

## 变化

- 读取每个挂载的 rclone `vfs/queue`、`vfs/stats` 和 `core/stats`，显示等待文件和真实远端传输进度。
- 新增传输中心，可在上传开始时自动打开且不抢占焦点。
- 仍有文件等待或正在上传时，取消挂载或退出会先警告。
- 刷新会清除旧缓存、主动刷新 VFS 目录，并通过直接远端列表核验结果。
- 连接卡片新增打开、刷新、传输和日志右键菜单。
- Settings 和同一 EXE 的命令行模式可以注册或移除 Windows 资源管理器刷新与传输命令。
- 资源管理器命令和重复启动请求会通过带认证的本机 IPC 转发给正在运行的 GUI。
- 每个平台和架构同时构建兼容原命名的 onefile 包与启动更快的 `-onedir.zip` 包。
