# SSH MountMate v0.4.2-alpha.4

This prerelease fixes settings persistence on fresh macOS installations. When no settings file
exists yet, the application now initializes the cache directory before validating the settings
form, so appearance, language, and other preferences can be saved immediately.

Ordinary settings saves no longer rewrite or remove the macOS login LaunchAgent unless the
"mount all at login" option actually changes. LaunchAgent failures include the affected plist path
while retaining the existing rollback behavior for real startup-setting changes.

Custom cache and transfer values are now shown directly in their selectors, for example
`Custom: 1h30m`. Existing rclone values outside the compact preset editor are kept as exact raw
values, preventing conversions such as `1h30m` to `1s` or `20G` to `20Mi`.

---

本预发行版修复了 macOS 全新安装时设置无法保存的问题。首次运行尚无设置文件时，程序会先
初始化默认缓存目录，再校验设置表单，因此外观、语言和其他偏好现在可以直接保存。

普通设置保存不再无条件改写或删除 macOS 登录 LaunchAgent；只有“登录时挂载全部连接”选项
确实发生变化时才会操作启动项。启动项失败信息会包含具体 plist 路径，而实际修改启动项失败时
仍保留原有回滚语义。

自定义缓存和传输参数现在直接在选择框中显示具体值，例如 `自定义：1h30m`。不属于紧凑预设
编辑器范围的既有 rclone 值会按原文保留，避免再把 `1h30m` 改成 `1s`，或把 `20G` 改成
`20Mi`。
