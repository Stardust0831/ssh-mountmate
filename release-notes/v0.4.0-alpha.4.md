# SSH MountMate v0.4.0-alpha.4

This prerelease fixes preview update discovery and a native SFTP host-key selection failure found
after `v0.4.0-alpha.3`.

## Update channels

- Prerelease builds now check all published, non-draft GitHub releases and select the highest
  semantic version, including newer alpha, beta, and release-candidate builds.
- A prerelease can also move forward to a newer stable release.
- Stable builds continue to accept stable releases only. A GitHub prerelease flag or prerelease
  version suffix excludes that release from the stable channel.
- Fixes GitHub asset decoding when the API returns both its internal asset URL and the public
  browser download URL. Automatic updates use only the validated public download URL.
- `v0.4.0-alpha.3` still uses the old stable-only endpoint, so one manual installation of this
  release is required. Future prerelease updates can then be discovered in the application.

## Native SFTP host-key selection

- When automatic `ssh-keyscan` produces no key, SSH MountMate now selects a known-hosts file only if
  it contains the requested host and port, instead of letting an unrelated managed file hide a
  matching user `~/.ssh/known_hosts` entry.
- OpenSSH hashed known-host entries remain eligible when no plaintext match can be established.
- Host-key validation remains enabled. The application does not silently trust unknown or changed
  server keys.
- The existing `--links` mount flag is unchanged. Historical log lines reporting missing symlink
  support are not the cause of the current host-key failure.

## Important prerelease limitations

- Packages are not yet production code-signed or notarized.
- This release does not include an installer or the planned Experimental macOS NFS backend.
- The Rust rewrite PR remains Draft and is not merged by this release.

本预发行版修复预览版更新发现，以及 `v0.4.0-alpha.3` 暴露的原生 SFTP 主机键文件选择问题。

- 预览版会从所有已发布、非 Draft 的 GitHub Release 中选择语义版本最高的版本，包括新的
  alpha、beta、RC，也允许升级到更新的正式版。
- 正式版仍只接收正式 Release；带预发行标记或预发行版本后缀的版本不会进入正式通道。
- 修复 GitHub API 同时返回内部资产 URL 和公开下载 URL 时的解析失败；自动更新只使用经过验证的
  公开下载 URL。
- `v0.4.0-alpha.3` 仍使用旧的正式版接口，因此需要手动安装本版本一次；之后的预览版可以在
  程序内继续发现更新。
- 当 `ssh-keyscan` 没有取得主机键时，程序只会选择包含目标主机和端口的 known_hosts 文件，
  不再让仅含其他主机键的 managed 文件遮蔽用户 `~/.ssh/known_hosts` 中已有的有效记录。
- 保留 OpenSSH 哈希主机记录兼容性，并继续强制主机键校验，不会静默信任未知或已变化的键。
- 挂载命令现有的 `--links` 参数保持不变；历史日志中的 symlink 报错不是本次失败原因。
- 本版本不包含安装器或实验性 macOS NFS 后端，PR 仍保持 Draft。
