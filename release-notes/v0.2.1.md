# SSH MountMate v0.2.1

Stable release for mount status, startup mount, bundled rclone, and capacity refresh improvements.

## Highlights

- Bundled rclone is materialized into a stable per-user app directory instead of a PyInstaller temporary directory.
- Windows login mount now uses the current-user Run key and the headless `--mount-startup-all` flow, avoiding `schtasks` access-denied failures.
- GUI mount status no longer exposes a stale/expired state. A running recorded rclone PID or ready mountpoint is treated as mounted.
- Windows unmount can continue when the rclone command line cannot be read but the recorded PID is still running.
- Capacity display now prefers local mounted filesystem capacity, avoiding repeated short-lived `rclone about` processes during normal refreshes.
- Local capacity refresh is lightweight and runs independently from the full mount-status scan.

## Fixes

- Detects managed bundled binaries named like `rclone-<hash>.exe` as a valid rclone dependency.
- Relaxes mount status fallback when command-line inspection is unavailable.
- Keeps remote capacity probes as a low-frequency fallback only when local mountpoint capacity cannot be read.
- Adds clearer startup setup error reporting and logs.
