# SSH MountMate v0.3.0-rc.2

Pre-release introducing truthful upload visibility, verified remote refresh, Explorer commands, and dual packaging.

## Changes

- Show each mount's real rclone VFS upload queue, remote bytes transferred, speed, and active-file progress.
- Add an optional automatically opened transfer center and warnings before unsafe unmount or exit actions.
- Verify refreshes by clearing the VFS cache, actively rereading the directory, and directly listing the remote.
- Add connection-card context menus and same-EXE Windows Explorer Refresh and Transfers commands.
- Forward second-launch and Explorer commands to the running GUI over authenticated loopback IPC.
- Publish both backward-compatible onefile packages and faster-starting `-onedir.zip` packages for all six targets.
- Preserve the selected onefile or onedir package type in update checks.
- Correct the build test to validate output paths on Windows as well as POSIX systems.

# SSH MountMate v0.3.0-rc.2 中文说明

这个预发布版新增真实上传可见性、远端刷新核验、资源管理器命令和双构建格式。

## 变化

- 显示每个挂载真实的 rclone VFS 上传队列、远端已传字节、速度和活动文件进度。
- 新增可自动打开的传输中心，并在不安全的取消挂载或退出操作前警告。
- 刷新会清除 VFS 缓存、主动重新读取目录并直接列出远端内容进行核验。
- 新增连接卡片右键菜单，以及由同一 EXE 处理的 Windows 资源管理器刷新和传输命令。
- 重复启动和资源管理器命令通过带认证的本机 IPC 转发给正在运行的 GUI。
- 六个平台目标同时发布兼容原命名的 onefile 包和启动更快的 `-onedir.zip` 包。
- 更新检查会保持用户当前选择的 onefile 或 onedir 包型。
- 修正构建测试，使输出路径断言同时适用于 Windows 和 POSIX 系统。
