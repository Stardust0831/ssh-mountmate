# SSH MountMate v0.2.5-rc.1

Pre-release focused on directory-cache freshness and safer default cache behavior.

## Changes

- Make the main Refresh button refresh rclone directory/metadata caches for currently mounted configs by calling `vfs/forget`, then refresh GUI status and capacity.
- Keep directory-cache refresh failures non-blocking; failed refresh details are written to `refresh.log`.
- Change default VFS cache mode to `minimal`, with clearer help text that it does not cache ordinary read-only file content.
- Set new cache defaults to 10G max cache size, 10G minimum free space, and 30s directory cache time.
- Only pass `--vfs-write-back` when VFS cache mode is `writes` or `full`; the setting is disabled in the UI for `off` and `minimal`.
- Warn before switching to write-cache modes where writes may complete locally before upload finishes.
- Migrate old default cache settings to the new safer defaults while preserving custom write-cache settings.

# SSH MountMate v0.2.5-rc.1 中文说明

这个预发布版主要调整目录缓存刷新和更稳妥的默认缓存行为。

## 变化

- 主界面的“刷新”现在会对已挂载配置调用 `vfs/forget`，刷新 rclone 的目录/元数据缓存，然后再刷新 GUI 状态和容量。
- 目录缓存刷新失败不会阻断整体刷新；失败详情会写入 `refresh.log`。
- 默认 VFS 缓存模式改为 `minimal`，并在帮助文案中明确它不会缓存普通只读文件内容。
- 新默认值改为：最大缓存大小 10G、最小剩余空间 10G、目录缓存时间 30s。
- 只有在 VFS 缓存模式为 `writes` 或 `full` 时才传递 `--vfs-write-back`；`off` 和 `minimal` 下 UI 会禁用该设置。
- 切换到写缓存模式前会弹出确认，提醒写入可能先在本地完成，随后继续上传到服务器。
- 旧版默认缓存设置会迁移到新的保守默认值；用户主动配置过的写缓存设置会保留。
