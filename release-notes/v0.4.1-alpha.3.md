# SSH MountMate v0.4.1-alpha.3

This prerelease fixes passphrase-protected SSH keys imported from OpenSSH configuration and makes
mount failures more compact without hiding their full diagnostics.

## Key passphrases and credential compatibility

- Native rclone mounts created from an SSH-config profile now pass the saved key passphrase
  alongside the resolved `IdentityFile`.
- Moving secrets to the system credential store retains the verified `rclone obscure` value as a
  compatibility copy instead of clearing it during settings save.
- A failed system-credential read remains an explicit error; the compatibility copy is retained to
  prevent data loss, not used as a silent fallback.

## Connection and failure UI

- Connection cards show a shorter mount-error summary and keep the full output in a dedicated log
  window available from the card.
- SSH-config connections using OpenSSH show their config path, Host alias, and command preview while
  treating resolved host/user/port values as an import snapshot.
- Windows no longer offers Interactive shared SSH for SSH-config and app-managed profiles, because
  Plink cannot preserve the complete OpenSSH configuration contract.

## Distribution notes

- This remains a prerelease, not a stable release.
- Update manifests and canonical assets retain Ed25519 verification.
- Packages are not Authenticode-signed or Apple Developer ID signed/notarized.

本预发行版修复从 OpenSSH 配置导入的加密私钥无法获得已保存短语的问题，并缩短主界面错误展示，
同时保留完整诊断入口。

## 密钥短语与凭据兼容性

- SSH config 来源的 Native rclone 挂载现在会把已保存的密钥短语与解析出的 `IdentityFile`
  一起传给 rclone。
- 将秘密迁移到系统凭据库时，会继续保留经过验证的 `rclone obscure` 兼容副本，不再在保存设置时清空。
- 系统凭据读取失败仍会明确报错；兼容副本用于防止数据丢失，不会被静默回退使用。

## 连接与错误界面

- 连接卡片只显示更短的挂载错误摘要，完整内容可从卡片打开独立日志窗口查看。
- 使用 OpenSSH 的 SSH config 连接会显示配置路径、Host 别名和命令预览；解析出的主机、用户和端口
  明确作为导入快照展示。
- Windows 不再为 SSH config 和应用托管配置提供交互式共享 SSH，因为 Plink 无法完整保留 OpenSSH
  配置语义。

## 分发说明

- 本版本仍是预发行版，不是正式稳定版。
- 更新 manifest 与规范资产继续使用 Ed25519 校验。
- 安装包尚未使用 Authenticode 或 Apple Developer ID 签名/公证。
