# SSH MountMate v0.4.1-alpha.1

This prerelease introduces a signed self-update trust root. The signing implementation itself does
not change mount, transfer, or server behavior; the package also carries the already completed
post-v0.4.0 opt-in desktop extensions listed below.

## Signed updates

- Verifies an Ed25519 signature before automatic installation. The signed manifest binds the
  version, stable/prerelease channel, and all six canonical ZIP names, sizes, and SHA-256 digests.
- Requires GitHub REST asset digests and metadata to independently match the signed manifest.
- Blocks automatic installation for a missing or invalid signature, unknown `key_id`, wrong
  version/channel/platform, changed asset, size mismatch, or SHA-256 mismatch. The release remains
  visible with the failure reason.
- Retains the existing GitHub host restrictions, safe ZIP extraction, detached updater, startup
  health confirmation, rollback, and stable/prerelease selection rules.
- Supports multiple embedded public keys for planned bridge-key rotation.

## Distribution notes

- Six canonical platform ZIPs are signed by one release manifest.
- Windows/Linux Releases still contain only canonical onefile packages; legacy onedir layouts do
  not have an automatic-update asset. macOS `.app` replacement remains supported.
- Existing v0.4.0 clients do not verify Ed25519 during the first update to this build. Users who
  need a strong bootstrap should manually verify the first signed package and published public-key
  fingerprint.
- Packages are not yet Authenticode-signed or Apple Developer ID signed/notarized.

## Included opt-in extensions since v0.4.0

- Experimental macOS rclone built-in NFS remains explicit opt-in; FUSE remains the default.
- System credential protection remains user-enabled rather than an automatic migration.
- Reusable interactive SSH sessions support the verified OpenSSH/Plink paths; one-time codes are
  never persisted.
- Windows and Linux mount commands and all server-side behavior remain unchanged.

本预发行版为自更新加入签名信任根。签名实现本身不修改挂载、传输或服务器行为；安装包也
包含下列已经完成的 v0.4.0 后续可选扩展。

## 签名更新

- 自动安装前验证 Ed25519 签名；签名 manifest 同时约束版本、正式/预览通道、六个平台 ZIP
  的名称、大小和 SHA-256。
- GitHub REST API 公布的附件摘要及元数据必须独立匹配签名 manifest。
- 签名缺失或错误、未知 `key_id`、版本/通道/平台错误、资产替换、大小或 SHA-256 不一致时，
  仍显示该更新及原因，但禁止自动安装。
- 保留既有 GitHub 域名限制、安全解压、独立 updater、启动健康确认、失败回滚和通道规则。
- 客户端支持多个公钥，为桥接式密钥轮换预留接口。

## 发行说明

- 一个签名 manifest 覆盖六个规范平台 ZIP。
- Windows/Linux Release 仍只有规范 onefile，旧 onedir 布局没有自动更新资产；macOS `.app`
  仍支持替换。
- v0.4.0 客户端首次升级到本版本时还不会验证 Ed25519。需要强首次信任的用户应手动核验首个
  签名包和另行公布的公钥指纹。
- 安装包尚未使用 Windows Authenticode 或 Apple Developer ID 签名/公证。

## 自 v0.4.0 起包含的可选扩展

- macOS rclone 内置 NFS 仍是明确的实验性可选项，FUSE 保持默认。
- 系统凭据保护仍需用户手动启用，不会自动迁移。
- 可复用交互式 SSH 会话覆盖已验证的 OpenSSH/Plink 路径；一次性验证码绝不保存。
- Windows/Linux 挂载命令和所有服务器端行为保持不变。
