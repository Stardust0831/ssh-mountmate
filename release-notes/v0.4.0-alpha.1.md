# SSH MountMate v0.4.0-alpha.1

This prerelease is the first full native Rust build of SSH MountMate. It is intended for testing
before the Rust rewrite is merged or promoted to a stable release.

## Native Rust application

- Replaces the Python/PyInstaller application with one Rust codebase on Windows, macOS, and Linux.
- Ships x64 and ARM64 builds without a Python runtime or extraction-based PyInstaller launcher.
- Keeps rclone as the verified mount/VFS engine.

## Onefile and onedir packages

- Plain `SSHMountMate-<platform>-<arch>.zip` assets contain one executable with the official rclone
  embedded and SHA-256 verified before compilation.
- `SSHMountMate-<platform>-<arch>-onedir.zip` assets contain the normal directory bundle; macOS
  onedir assets contain the native `.app` bundle.
- Embedded rclone is materialized on first use as a content-addressed managed copy outside the
  application, so GUI restart and self-update do not stop active mounts.
- Self-update selects the current package type, verifies GitHub's published digest and size,
  stages replacement beside the installation, performs an authenticated health handshake, and
  rolls back after a failed restart.

## Mounting and transfer behavior

- Preserves manual, SSH config, batch import, SAI, native SFTP, and external OpenSSH connection
  paths.
- Preserves host-key validation and reports changed keys instead of silently disabling checks.
- Reports queued VFS writes truthfully and shows per-connection bottom-right transfer windows plus
  a complete transfer center.
- Keeps existing mounts and queued uploads alive while the GUI package is updated.
- Adds verified Explorer/Finder/Linux file-manager refresh commands.

## Desktop integration

- Native Windows Toast, taskbar progress, tray, Explorer menu, and login startup integration.
- Native macOS application bundle, notifications, menu bar, Dock progress, Finder service, and
  login startup integration.
- Linux X11/Wayland notification and tray capability handling plus common file-manager actions.

## Important prerelease limitations

- Packages are not yet production code-signed or notarized.
- macOS still uses the existing FUSE-T/macFUSE-compatible mount path. The optional Experimental
  `rclone nfsmount` backend is planned for the next development stage and is not included here.
- The Rust rewrite PR remains Draft and is not merged by this release.
