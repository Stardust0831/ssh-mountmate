# SSH MountMate v0.2.1-rc.2

This prerelease removes the legacy CLI command layer so future fixes target the GUI-backed application path.

## What's Changed

- Remove the unused legacy `core.py` CLI commands (`mount`, `umount`, `sync`, `status`, `doctor`, etc.).
- Keep the shared SSH config, rclone config, lock, path, and platform helpers used by the GUI.
- Confirm `python -m ssh_mountmate` and the packaged executable continue to use the GUI entrypoint.
- Add entrypoint tests so the old core CLI does not accidentally return.

# SSH MountMate v0.2.1-rc.2 中文说明

这个预发布版移除了旧的 CLI command 层，让后续修复都集中在 GUI 实际使用的应用路径上。

## 主要更新

- 移除未使用的旧 `core.py` CLI 命令层，包括 `mount`、`umount`、`sync`、`status`、`doctor` 等。
- 保留 GUI 仍在使用的 SSH config、rclone config、锁、路径和平台辅助函数。
- 确认 `python -m ssh_mountmate` 和打包后的可执行程序仍然使用 GUI 入口。
- 新增入口测试，避免旧 core CLI 意外回归。
