# SSH MountMate v0.4.2-alpha.3

This prerelease strengthens Windows folder-mount validation. A custom folder mountpoint must now
be backed by a normal local NTFS drive root, and every existing parent component is checked for
reparse points. Mounted volumes, junction-backed paths, remote filesystems, and fixed-advertising
WinFsp filesystems are rejected before saving instead of failing after rclone starts.

Interactive shared SSH now allows simple resolved OpenSSH-config profiles to be selected while
still rejecting ProxyJump and ProxyCommand at session creation. Modern OpenSSH private keys are no
longer passed to Plink, which cannot read that format; password and 2FA authentication continue in
the app terminal. Active mounts keep ownership of their shared SSH session, so Retry and End are
disabled until the connection is unmounted. Retrying an inactive session creates a keyed fresh
terminal state.

Windows rclone mount commands now request rclone's native no-console mode in addition to hidden
process creation flags, preventing short-lived failure consoles from appearing. Full failure logs
remain available only through the explicit in-app log viewer.

---

本预发行版加强了 Windows 文件夹挂载点检查。自定义文件夹挂载点现在必须位于普通的本地
NTFS 盘符根目录下，并会逐级检查现有父路径中的重解析点。卷挂载文件夹、junction 路径、
远程文件系统和伪装成固定盘的 WinFsp 文件系统会在保存前被拒绝，而不是等 rclone 启动后失败。

简单且已解析的 OpenSSH config 配置现在可以选择交互式共享 SSH，但创建会话时仍会拒绝
ProxyJump 和 ProxyCommand。Plink 无法读取的新版 OpenSSH 私钥不再传给 Plink，密码和 2FA
仍在应用内终端完成。挂载活动期间共享 SSH 会话由挂载持有，因此必须先卸载才能重试或结束；
非活动会话重试时会创建带新标识的全新终端状态。

Windows rclone 挂载命令现在除了隐藏进程创建标志外，还会请求 rclone 自身的无控制台模式，
避免挂载失败时出现短暂的独立日志窗口。完整失败日志仍只通过应用内的“查看完整日志”打开。
