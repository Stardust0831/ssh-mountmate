# SSH MountMate v0.4.3

This release replaces single-folder connection organization with flexible tags and adds safer,
more efficient batch management.

- Connections can have multiple tags. The tag row below Search filters the list quickly, tagged
  connections appear before untagged ones, and existing folder values migrate to tags without
  losing compatibility with older configuration files.
- Batch mode supports visible select-all, per-tag selection, mount, unmount, tag assignment, and
  confirmed deletion. Custom ordering remains a separate mode and preserves the tagged/untagged
  grouping.
- Login auto-mount remains configurable per connection in the collapsed Settings connection panel.
  Tag, ordering, deletion, and Settings writes are serialized so an older asynchronous snapshot
  cannot overwrite a newer change.
- When settings cannot be read, the recovery dialog now names the exact original settings path and
  the exact saved or attempted `.invalid...bak` path. It preserves raw read, write, cleanup, and
  restore errors, explains permission, lock, path, and disk-space failures, and identifies the
  in-memory defaults and cache directory in use.
- If an unreadable settings file cannot be backed up, startup leaves it untouched and performs no
  automatic settings write. Saving remains an explicit user action after the recovery dialog has
  been reviewed.

本版本以灵活的多标签取代单一文件夹归类，并增加更安全、高效的批量连接管理。

- 每个连接可以拥有多个标签。搜索框下方的标签栏可快速筛选连接；有标签连接优先显示。已有文件夹
  数据会迁移为标签，同时继续兼容旧版配置文件。
- 批量模式支持对当前可见连接全选、按标签选择、批量挂载、卸载、添加标签和确认删除。自定义排序
  保持为独立模式，并继续保证有标签连接位于无标签连接之前。
- 登录自动挂载仍可在“设置”中默认折叠的连接面板里逐连接配置。标签、排序、删除与设置保存会串行
  执行，旧的异步快照不会覆盖较新的修改。
- 设置无法读取时，恢复弹窗会明确列出原设置路径，以及实际保存或尝试使用的 `.invalid...bak`
  备份路径。弹窗保留读取、写入、清理和恢复原文件的系统错误，区分权限、锁占用、路径和磁盘空间
  问题，并说明当前使用的内存默认值与缓存目录。
- 如果无法备份不可读的设置文件，启动过程会保持原文件不变，也不会自动写入设置。用户查看恢复
  弹窗后，仍可通过主动点击“保存”来重试。
