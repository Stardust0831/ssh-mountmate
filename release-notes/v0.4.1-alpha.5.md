# SSH MountMate v0.4.1-alpha.5

This prerelease replaces external interactive SSH consoles with terminals owned and rendered by
SSH MountMate. It is intended for host-level validation of password prompts, key passphrases,
2FA, host-key confirmation, and other interactive authentication flows.

## App-managed interactive SSH

- Linux and macOS interactive OpenSSH sessions now run in an application terminal backed by a
  real PTY. Windows direct manual connections use the packaged verified Plink through ConPTY.
- A mount requested during authentication is queued once and resumes automatically when the
  shared SSH session is ready. Repeated clicks do not create duplicate terminals or mounts.
- Terminals can be hidden and reopened without ending their session. Retry and End session remain
  explicit actions, and exiting the app warns when live terminal sessions will be terminated.
- OpenSSH masters remain attached to the app PTY rather than entering `ControlPersist` background
  mode, so the application owns the complete session lifetime.

## Safety and compatibility

- Terminal input and output stay in memory and are not written to application logs. Diagnostic
  events redact terminal payloads.
- Exact executable paths and arguments are retained without shell scripts or lossy Unicode
  conversion.
- Saving a connection cannot race a queued login into mounting stale connection settings.
- Interactive readiness is checked before rclone resolution, and Unix control sockets retain
  owner, type, permission, symlink, stale-socket, and short-path validation.
- Existing settings, credentials, mounts, and non-interactive SSH transports remain compatible.

## Distribution notes

- This remains a prerelease, not a stable release.
- Update manifests and canonical assets retain Ed25519 verification.
- Packages are not Authenticode-signed or Apple Developer ID signed/notarized.

本预发行版用 SSH MountMate 自己管理和渲染的终端替代外部交互式 SSH 控制台，主要供宿主机上
验收密码提示、私钥短语、2FA、主机密钥确认及其他交互式认证流程。

## 应用内交互式 SSH

- Linux 和 macOS 的交互式 OpenSSH 会话现在运行在应用内真实 PTY 终端中；Windows 手动直连
  使用随包提供且经过校验的 Plink 和 ConPTY。
- 认证期间发起的挂载只排队一次，共享 SSH 会话就绪后会自动继续。重复点击不会创建重复终端
  或重复挂载。
- 终端隐藏后可重新打开且不会结束会话；重试和结束会话仍是显式操作。退出应用时，如果仍有
  活跃终端会话，程序会给出确认提示。
- OpenSSH master 保持连接在应用 PTY 上，不再通过 `ControlPersist` 转入后台，因此完整生命周期
  由应用持有。

## 安全与兼容性

- 终端输入输出仅保留在内存，不写入应用日志；诊断事件会隐藏终端载荷。
- 可执行文件路径和参数保持原值，不使用 shell 脚本，也不进行有损 Unicode 转换。
- 保存连接时，不会与排队中的登录竞争并按旧连接设置发起挂载。
- 程序会在解析 rclone 前检查交互会话是否就绪；Unix 控制套接字继续校验所有者、类型、权限、
  符号链接、陈旧套接字和长路径回退。
- 现有设置、凭据、挂载和非交互式 SSH 传输保持兼容。

## 分发说明

- 本版本仍是预发行版，不是正式稳定版。
- 更新 manifest 与规范资产继续使用 Ed25519 校验。
- 安装包尚未使用 Authenticode 或 Apple Developer ID 签名/公证。
