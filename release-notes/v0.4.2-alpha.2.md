# SSH MountMate v0.4.2-alpha.2

This prerelease fixes excessive vertical whitespace around the batch mount actions and connection
search controls introduced in `v0.4.2-alpha.1`. Responsive controls now shrink to their content
height while retaining narrow-window wrapping and all four text-size presets.

Windows background mounts no longer combine incompatible console creation flags. OpenSSH and
shared-Plink SFTP connectors are launched through an app-owned, no-console proxy with exact
standard-stream forwarding, so rclone connector processes stay inside the application workflow
without flashing a system terminal. Resolved SSH-config profiles with HostName, User, and Port can
now use interactive shared SSH even when the profile has no IdentityFile; password and 2FA prompts
remain in the app terminal, and readiness waits for the shared session to be genuinely available.

---

本预发行版修复了 `v0.4.2-alpha.1` 中“全部挂载”操作区和连接搜索控件前后出现过多纵向空白的
问题。响应式控件现在按内容高度收缩，同时保留窄窗口自动换行和四档字号支持。

Windows 后台挂载不再组合互相冲突的控制台创建标志。OpenSSH 和共享 Plink 的 SFTP connector
现在通过应用管理的无控制台代理启动，并精确转发标准输入输出，因此不会再闪出系统终端。
已经解析出 HostName、User 和 Port 的 SSH config 配置现在也可使用交互式共享 SSH，即使配置中
没有 IdentityFile；密码和 2FA 提示仍留在应用终端内，就绪检查会等待共享会话真正可用。
