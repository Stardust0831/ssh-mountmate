# SSH MountMate v0.4.1-alpha.6

This prerelease fixes in-app update checks failing when GitHub's shared anonymous REST API quota is
exhausted.

## Update reliability

- Prerelease update checks still use the GitHub Releases API when it is available.
- If that request is rate-limited or unavailable, SSH MountMate discovers published release tags
  through GitHub's public Atom feed instead of retrying the same exhausted API quota.
- The fallback only exposes an automatic update after the fixed-tag manifest and signature pass
  the embedded Ed25519 trust policy. Canonical repository URLs, release version and channel, the
  complete six-platform asset set, exact size, and SHA-256 remain enforced.
- CI now exercises the REST channel, the Atom fallback, and the public application update entry
  point against published signed assets.

## Included changes

- Includes the app-managed PTY/ConPTY interactive SSH terminal and mount-resume fixes from
  `v0.4.1-alpha.5`.
- Existing settings, credentials, mounts, and packaged rclone behavior remain compatible.

本预发行版修复了 GitHub 共享匿名 REST API 配额耗尽时，应用内更新检查直接失败的问题。

## 更新可靠性

- GitHub Releases API 可用时，预发行更新检查仍优先使用该接口。
- 请求受到限流或接口不可用时，SSH MountMate 会通过 GitHub 的公开 Atom feed 发现已发布版本，
  不再重复请求已经耗尽配额的同一接口。
- 备用路径只有在固定 tag 的 manifest 和签名通过内置 Ed25519 信任策略后，才会提供自动安装。
  规范仓库 URL、版本和发布通道、完整六平台资产集合、精确大小及 SHA-256 校验均保持不变。
- CI 现在会针对已发布且已签名的资产分别验证 REST 通道、Atom 备用路径和应用公开更新入口。

## 包含的改动

- 包含 `v0.4.1-alpha.5` 的应用内 PTY/ConPTY 交互式 SSH 终端与挂载自动继续修复。
- 现有设置、凭据、挂载和随包 rclone 行为保持兼容。
