# SSH MountMate v0.4.1-alpha.2

This prerelease prioritizes safe credential migration, actionable mount failures, and Windows
mountpoint preflight. It is published after automated platform gates so existing signed builds can
install it through the app updater; broader interactive-terminal and appearance work will continue
in later prereleases.

## Credential safety

- System credential migrations now preserve the previous entry when write/read verification fails.
- Migration records are saved and reloaded in staged form before the old representation is retired.
- Returning to `rclone obscure` saves and reloads the replacement before system entries are deleted.
- Blank secret inputs explicitly mean keep; the editor shows where a secret is stored and provides
  separate replace and clear actions without displaying secret plaintext.

## Mount diagnostics

- The last mount failure remains on its connection card until dismissed, retried, or superseded.
- Failure cards provide direct retry and mount-log actions.
- The log window now explains when a connection must be selected and shows the expected path when a
  log has not been created.
- Windows custom folder mountpoints reject UNC, mapped network, missing, and unsupported backing
  volumes before rclone starts; fixed local volumes and drive-letter mounts remain supported.

## Distribution notes

- This remains a prerelease, not a stable release.
- Update manifests and canonical assets retain the Ed25519 verification introduced in alpha.1.
- Packages are not Authenticode-signed or Apple Developer ID signed/notarized.

## 凭据安全

- 系统凭据写入或回读验证失败时会恢复迁移前的条目，不再盲目删除确定性引用。
- 迁移候选记录必须保存并从磁盘重载验证，之后才会清理旧表示。
- 切回 `rclone obscure` 时，会先保存并重载新表示，最后才删除系统凭据条目。
- 空白秘密输入明确表示保留；编辑器显示秘密存储位置，并提供独立的替换与清除操作，绝不显示明文。

## 挂载诊断

- 最近一次挂载失败会保留在对应连接卡片，直到关闭、成功重试或由新操作取代。
- 错误卡片可直接重试或打开该连接的挂载日志。
- 日志窗口会提示先选择连接；日志尚未创建时会显示预期路径和原因。
- Windows 自定义目录会在启动 rclone 前拒绝 UNC、映射网络盘、缺失或不支持的底层卷；固定本地卷和
  盘符挂载保持可用。

## 分发说明

- 本版本仍是预发行版，不是正式稳定版。
- 更新 manifest 与六个平台规范资产继续使用 alpha.1 引入的 Ed25519 校验。
- 安装包尚未使用 Authenticode 或 Apple Developer ID 签名/公证。
