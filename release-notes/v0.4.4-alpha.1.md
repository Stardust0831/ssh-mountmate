# SSH MountMate v0.4.4-alpha.1

This prerelease refines connection organization and makes portable self-updates recover from an
interrupted previous transaction.

- The prominent organization action now opens dedicated tag management. Select one or more
  connections to assign tags, remove a tag from one connection, or remove a tag globally.
- Batch operations focus on operational work: mount, unmount, enable or disable login auto-mount,
  and confirmed connection deletion. Per-tag selection appears only after entering batch mode.
- Custom ordering is available from the compact control beside the sort selector. Reordering is a
  draft until **Save order** is pressed; Cancel restores the previous order, and tags no longer
  force connections into separate ordering groups.
- Tags are visually quieter and limited to 8 per connection and 24 Unicode characters each. The
  UI, connection validation, and storage layer enforce the same limits.
- Login auto-mount reports interactive SSH connections that were skipped because they require a
  live login session, while preserving successfully saved preferences if OS startup integration
  must be retried.
- Portable updates no longer remain blocked by a leftover `.ssh-mountmate-backup` transaction. If
  the current installation is valid, the old backup is preserved under a unique recovered name
  before the next update begins.

本预发布版优化了连接整理交互，并允许便携版自更新从上一次中断的更新事务中自动恢复。

- 主页原本醒目的整理按钮现在进入独立的“标签管理”。可以选择一个或多个连接批量添加标签、从单个
  连接移除标签，或全局删除某个标签。
- “批量操作”专注于挂载、卸载、启用或禁用登录自启，以及经过二次确认的连接删除。按标签批量选择
  仅在进入批量模式后显示。
- 自定义排序入口收进排序选择器旁的小按钮。调整只修改排序草稿，点击“保存排序”才会写入；取消会
  恢复原顺序，标签也不再强制把连接划分为不同排序区域。
- 标签视觉权重降低，并限制为每个连接最多 8 个、每个标签最多 24 个 Unicode 字符。界面、连接校验
  和存储层使用相同限制。
- 批量登录自启会明确提示因需要实时登录会话而被跳过的交互式 SSH 连接；如果系统自启集成需要稍后
  重试，已经保存的连接偏好仍会保留。
- 便携版更新不再被遗留的 `.ssh-mountmate-backup` 更新事务永久阻塞。当前安装有效时，旧备份会先被
  重命名到唯一的恢复路径并保留，然后再开始下一次更新。
