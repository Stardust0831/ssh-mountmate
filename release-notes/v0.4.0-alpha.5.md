# SSH MountMate v0.4.0-alpha.5

This prerelease improves Explorer refresh reporting, concurrent transfer visibility, log selection,
and capacity status on top of `v0.4.0-alpha.4`.

## Refresh behavior

- Fixes Windows Explorer subdirectory paths ending in `\\.` so nested folder refreshes target the
  intended remote directory.
- Clarifies that refresh runs `vfs/forget` and `vfs/refresh`, while the reported number is the
  directory's current direct-child count rather than a count of invalidated cache entries.
- Treats a missing `operations/list` result as an error instead of silently reporting zero. A real
  empty directory can still correctly report zero direct entries.

## Transfer window and progress

- Replaces per-mount transfer popups with one shared always-on-top window for all active mounts.
- Keeps the window compact by default and allows it to expand to show each mount and file transfer.
- Restores normal window decorations and Windows activation behavior so the transfer window can be
  dragged.
- Shows transfer progress only while work is queued, uploading, or in an error state. VFS cache
  exhaustion is reported as an error instead of false completion.

## Logs and capacity

- Makes mount logs selectable with mouse and keyboard. Native copy shortcuts work, and the Copy log
  button copies the selection when present or the complete loaded log otherwise.
- Keeps the capacity bar visible for every mounted connection, including explicit checking and
  unknown states.
- Starts capacity discovery immediately after a mount and schedules a follow-up query when multiple
  mounts complete concurrently.

## Important prerelease limitations

- Packages are not yet production code-signed or notarized.
- This release does not include an installer or the planned Experimental macOS NFS backend.
- The Rust rewrite PR remains Draft and is not merged by this release.

本预发行版在 `v0.4.0-alpha.4` 基础上改进 Explorer 刷新提示、并发传输窗口、日志选择和容量状态。

- 修复 Windows Explorer 传入以 `\\.` 结尾的子目录路径，确保刷新目标是正确的远端目录。
- 明确刷新会执行 `vfs/forget` 和 `vfs/refresh`；显示的数字是刷新后目录当前的直接子条目数量，
  不是失效缓存条目数量。缺少 `operations/list` 数据时会报错，不再静默显示为零。
- 所有挂载复用一个置顶传输窗口。窗口默认紧凑，可展开查看每个挂载和具体文件，并恢复标准窗口
  装饰及可拖动行为。
- 只在排队、上传或错误状态下显示传输进度；VFS 缓存空间不足不会被显示为已完成。
- 挂载日志支持鼠标和键盘选择、原生复制快捷键；复制按钮优先复制选区，否则复制全部已加载日志。
- 容量进度条在挂载后常驻，明确显示查询中、未知或实际容量；挂载完成后立即查询，并处理多个挂载
  同时完成时的后续查询。
- 本版本不包含安装器或实验性 macOS NFS 后端，PR 仍保持 Draft。
