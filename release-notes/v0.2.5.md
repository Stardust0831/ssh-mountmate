# SSH MountMate v0.2.5

Release focused on VFS cache behavior, directory freshness, and connection security.

## Changes

- Make the main Refresh button clear rclone directory and metadata caches for mounted configs before refreshing GUI status and capacity.
- Use the recommended default cache profile: `--vfs-cache-mode full`, `--vfs-cache-max-age 30m`, `--vfs-write-back 0s`, and `--dir-cache-time 5m`.
- Keep cache size and minimum free space unlimited unless the user explicitly configures them.
- Preserve custom cache settings while migrating earlier default profiles to the recommended profile.
- Skip unreadable `known_hosts_file` paths, avoiding access-denied mount failures on locked-down Windows profiles.
- Pin host keys after their first scan instead of replacing trusted entries on every mount.
- Stop on a host key mismatch instead of retrying with host key validation disabled.
- Write application configuration and rclone credential files atomically with owner-only permissions on POSIX systems.
- Run unit tests and Python compilation checks before CI build and release packaging.

# SSH MountMate v0.2.5 中文说明

这个版本主要调整 VFS 缓存、目录刷新和连接安全性。

## 变化

- 主界面的“刷新”会先清除已挂载配置的 rclone 目录和元数据缓存，再刷新界面状态与容量。
- 使用推荐的默认缓存策略：`--vfs-cache-mode full`、`--vfs-cache-max-age 30m`、`--vfs-write-back 0s` 和 `--dir-cache-time 5m`。
- 除非用户主动配置，否则不限制缓存大小和最小剩余空间。
- 把早期默认缓存策略迁移到新的推荐策略，同时保留用户自定义缓存设置。
- `known_hosts_file` 不可读时跳过该路径，避免 Windows 受限用户目录导致拒绝访问和挂载失败。
- 首次扫描 host key 后固定使用，不再在每次挂载时覆盖已经信任的记录。
- host key 不匹配时停止挂载，不再关闭 host key 校验重试。
- 以原子方式写入应用配置和 rclone 凭据文件，并在 POSIX 系统上限制为仅所有者可读写。
- CI 构建和发布打包前运行单元测试与 Python 编译检查。
