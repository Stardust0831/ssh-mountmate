# SSH MountMate v0.4.0-alpha.7

This prerelease improves transfer safety, settings usability, log access, and version replacement.

## Highlights

- Mount logs now open in a dedicated window with connection selection, partial-text copying, refresh,
  and automatic positioning at the newest line.
- Connection cards are back to the four primary actions. Remote paths and mountpoints use the
  structured choices familiar from the earlier desktop UI.
- Cache limits and timing settings use compact presets with a digits-only custom value dialog and
  unit selector. Existing non-preset values are preserved until explicitly edited.
- Isolated rclone RC polling failures retain the last valid transfer snapshot. The unavailable state
  appears only after three consecutive failures.
- Locally queued files are described as waiting for write-back or an upload slot instead of showing a
  misleading zero-speed transfer.
- Closing transfer views never cancels uploads. Pending or unknown uploads now guard unmount, with
  wait-for-sync as the preferred action and a separate immediate-unmount risk confirmation.
- Starting a different build no longer silently activates an older tray instance. After identity
  verification, the user can gracefully replace the old interface while existing rclone mounts keep
  running.

## Important limitations

- Packages are prerelease builds and are not production code-signed or notarized.
- This release does not include an installer, server changes, or the planned Experimental macOS NFS
  backend. The Rust rewrite PR remains Draft.

本预发行版改进传输安全、设置易用性、日志查看和新旧版本替换行为。

- 挂载日志改为独立窗口，可选择连接、刷新、选中局部文本复制，并默认定位到最新一行。
- 主页面恢复四个主要按钮；远端路径和挂载点恢复旧版的结构化下拉选择。
- 缓存大小、寿命、剩余空间、写回、目录缓存和读取缓冲改为精简预设；自定义值使用纯数字
  输入和单位下拉框，旧的非预设值不会被静默改写。
- 单次 rclone RC 轮询失败会保留上一次有效传输状态，连续三次失败后才显示不可用。
- 尚未开始上传的文件明确显示为等待写回延迟或上传槽位，不再用零速率造成误解。
- 关闭传输界面不会取消上传。仍有待上传内容或状态未知时，取消挂载会优先提供“同步完成后
  自动取消挂载”，立即取消挂载需要再次确认风险。
- 打开不同版本或路径的 EXE 时不再静默激活旧托盘实例；验证进程身份后可优雅替换界面，
  已有 rclone 挂载继续运行。
- 本版本不包含安装器、服务器修改或实验性 macOS NFS 后端，PR 仍保持 Draft。
