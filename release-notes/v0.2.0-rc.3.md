# SSH MountMate v0.2.0-rc.3

This prerelease tightens mount process handling and adds focused tests around mount readiness and process matching.

## What's Changed

- Close the parent GUI/CLI log-file handle immediately after starting `rclone mount`, preventing long-lived handle leaks.
- Use rclone process command-line matching on Windows, macOS, and Linux instead of relying only on a saved PID.
- Keep mount status detection tolerant enough for normal use while making process termination stricter to avoid killing unrelated PID-reused processes.
- Replace the old Windows CLI `sleep(2)` mount check with readiness polling shared with the GUI mount path.
- Add unit tests for rclone command matching, procfs process discovery, mount status decisions, and mount readiness polling.

# SSH MountMate v0.2.0-rc.3 中文说明

这个预发布版加强了挂载进程处理，并为挂载 ready 判断和进程匹配补了针对性测试。

## 主要更新

- 启动 `rclone mount` 后立即关闭 GUI/CLI 父进程持有的日志文件句柄，避免长期句柄泄露。
- Windows、macOS 和 Linux 都会读取 rclone 进程命令行，不再只依赖保存的 PID 判断挂载状态。
- 挂载状态显示保持适度容错；真正终止进程时使用更严格的匹配，避免 PID 复用导致误杀。
- 旧 Windows CLI mount 路径不再使用固定 `sleep(2)`，改为和 GUI 共用 readiness 轮询。
- 新增单元测试，覆盖 rclone 命令匹配、procfs 进程发现、挂载状态判断和 readiness 轮询。
