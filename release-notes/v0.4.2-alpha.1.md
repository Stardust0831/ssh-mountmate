# SSH MountMate v0.4.2-alpha.1

This prerelease improves custom mountpoint safety, interactive SSH responsiveness, connection
organization, and interface accessibility.

## Mountpoints and interactive SSH

- Custom mountpoints are checked while editing and again before saving. Unsupported Windows UNC,
  mapped-network, removable, missing, occupied, or otherwise invalid targets block saving with a
  concrete error before credentials or managed SSH files are changed.
- Windows interactive SSH continues to use the verified packaged Plink inside the app-owned ConPTY
  terminal. Shared-session readiness probes now run off the UI thread, have bounded termination,
  suppress standard streams, and use `CREATE_NO_WINDOW` so they cannot repeatedly flash consoles.
- Late readiness results cannot revive an exited terminal or resume a mount more than once.

## Organization and appearance

- Named connection folders always appear before Uncategorized in saved, name, and host order.
- Settings provide Small, Standard, Large, and Extra large text sizes. Changes preview immediately,
  apply to every app window including the SSH terminal, restore on Cancel, and persist on Save.
- The main toolbar and search/sort controls wrap at narrow effective widths, and primary windows now
  have accessible minimum sizes for larger text.

本预发行版改进了自定义挂载点安全性、交互式 SSH 响应、连接整理和界面可访问性。

## 挂载点和交互式 SSH

- 自定义挂载点会在编辑时和保存前分别检查。Windows 上的 UNC、映射网络盘、可移动卷、缺失、
  已占用或其他无效目标会显示具体错误并阻止保存，且检查发生在修改凭据或托管 SSH 文件之前。
- Windows 交互式 SSH 继续在应用管理的 ConPTY 终端中使用经过校验的随包 Plink。共享会话就绪
  探针已移出 UI 线程，具备有界终止、关闭标准流并使用 `CREATE_NO_WINDOW`，不会反复闪出控制台。
- 过期的就绪结果不会复活已退出的终端，也不会让同一次挂载重复继续。

## 整理和外观

- 命名文件夹在保存顺序、名称和主机排序下都始终显示在“未分类”之前。
- 设置新增小、标准、大、特大四档字号；切换时立即预览，覆盖包括 SSH 终端在内的所有窗口，
  取消时恢复，保存后持久生效。
- 主工具栏和搜索/排序控件会在有效宽度较窄时自动换行，主要窗口也为大字号设置了合理最小尺寸。
