# SSH MountMate v0.2.3

Stable release for connection reliability, onboarding, settings help, and Add config UI polish.

## Highlights

- Improve SFTP mount reliability for host-key handling, Windows SSH file permissions, copied private keys, and remote symlinks.
- Show an empty-state prompt with an Add config button when no mount configs exist.
- Split documentation into English `README.md` and Chinese `README.zh-CN.md`.
- Keep batch mount concurrency fixed at 4 workers and batch unmount concurrency fixed at 8 workers.
- Move settings and Add config explanations onto dedicated `?` help icons.
- Clarify VFS cache settings, including `default (no size limit)` for max cache size.
- Fix language switching so the main mount list does not disappear.
- Improve connection-card disabled-button feedback by showing the reason when clicked.
- Rework Settings and Add config scrolling so action buttons remain reachable.
- Clean up Add config spacing, required-field markers, and tooltip positioning.
- Refine batch SSH config import with Import/Overwrite checkboxes, bulk selection, and clearer duplicate handling.

## Fixes

- Normalize automatically scanned host keys to the marker format used by rclone.
- Retry once without rclone `known_hosts_file` when rclone reports a known-host mismatch.
- Repair Windows ACLs for app-managed SSH config files and copied private keys.
- Pass `--links` to `rclone mount` so remote symlinks do not abort mounts.
- Avoid oversized forced font scaling while retaining better readability.
- Fix clipped batch import checkboxes.
