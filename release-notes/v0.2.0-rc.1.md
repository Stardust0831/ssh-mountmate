# SSH MountMate v0.2.0-rc.1

This prerelease keeps the single-executable packaging model while fixing the Windows PyInstaller temporary-directory cleanup warning reported in #3.

## What's Changed

- Keep PyInstaller `onefile` packaging for release builds.
- Materialize the bundled rclone binary into a stable managed bin directory before starting mounts.
- Use hash-named managed rclone copies so active mounts are not overwritten during upgrades.
- Clean up old managed rclone copies on Windows, macOS, and Linux while preserving copies still referenced by running rclone processes.
- Move the Windows managed rclone directory to `%LOCALAPPDATA%\SSHMountMate\bin`.
- Use `~/Library/Application Support/SSHMountMate/bin` for managed rclone copies on macOS.
- Keep Linux managed rclone copies under `${XDG_DATA_HOME:-~/.local/share}/ssh-mountmate/bin`.

## Notes

- This is an alternative to the `onedir` packaging workaround proposed in #4.
- Thanks to @bane-dysta for reporting the Windows `_MEI*` cleanup warning and proposing the packaging workaround.

# SSH MountMate v0.2.0-rc.1 中文说明

这个预发布版在保留单文件可执行程序打包方式的同时，修复 #3 中报告的 Windows PyInstaller 临时目录清理警告。

## 主要更新

- Release 构建继续使用 PyInstaller `onefile` 单文件模式。
- 启动挂载前，会先把内置 rclone 复制到稳定的托管 bin 目录。
- 托管 rclone 使用带哈希的文件名，升级时不会覆盖正在被活跃挂载使用的旧 rclone。
- Windows、macOS 和 Linux 都会清理旧的托管 rclone 副本，同时保留仍被运行中 rclone 进程引用的副本。
- Windows 托管 rclone 目录改为 `%LOCALAPPDATA%\SSHMountMate\bin`。
- macOS 托管 rclone 目录使用 `~/Library/Application Support/SSHMountMate/bin`。
- Linux 托管 rclone 目录保持在 `${XDG_DATA_HOME:-~/.local/share}/ssh-mountmate/bin`。

## 注意

- 这是对 #4 中 `onedir` 打包规避方案的替代实现。
- 感谢 @bane-dysta 报告 Windows `_MEI*` 清理警告，并提出打包方式调整方案。
