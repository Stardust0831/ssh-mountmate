# SSH MountMate v0.4.0-alpha.2

This prerelease improves cached-upload responsiveness, transfer reporting, capacity discovery, and
reduces the public package matrix from twelve ZIPs to six.

## Upload and file-manager responsiveness

- Restores the recommended VFS write-back delay to 5 seconds for the recognizable previous default
  profile. This gives close, rename, and metadata operations a local-cache window before upload and
  reduces the chance that a large copy makes the file manager appear stuck.
- Keeps custom cache profiles unchanged; the new value applies on the next mount and does not
  interrupt existing mounts.
- Combines `core/stats` transfer details with the VFS disk-cache upload state when `vfs/queue`
  temporarily omits an active file, making the bottom-right upload window more reliable.
- The displayed progress continues to represent the remote upload, not merely completion of the
  operating system's local copy dialog.

## Capacity reporting

- Adds a non-interactive remote `df -Pk` fallback when local mount statistics, Lustre project quota,
  and `rclone about` cannot provide usable capacity information.
- Password profiles without a safe managed SSH route do not trigger an unexpected password prompt.

## Six canonical downloads

- Windows x64/ARM64 and Linux x64/ARM64 use onefile ZIPs with verified official rclone embedded.
- macOS x64/ARM64 use native `.app` ZIPs with rclone included in the application bundle.
- Separate `-onedir.zip` assets are no longer published, reducing the release from twelve ZIPs to six.
- The updater now selects the single canonical asset for the current OS and CPU architecture.
- Users running a noncanonical alpha.1 package should manually download and install the matching
  alpha.2 package once. The retired alpha.1 onedir assets had no recorded downloads when this
  decision was made.

## Important prerelease limitations

- Packages are not yet production code-signed or notarized.
- macOS ARM64 still has the documented active-upload update timing risk from alpha.1. Treat updating
  the GUI during an active upload on macOS ARM64 as experimental.
- macOS continues to use the existing FUSE-T/macFUSE-compatible mount path. The optional
  Experimental `rclone nfsmount` backend is not included in this release.
- The Rust rewrite PR remains Draft and is not merged by this release.

本预发行版改善了写缓存下的文件管理器响应、真实上传进度和容量探测，并把公开发行包从十二个
ZIP 收敛为六个。

- 旧版可识别的默认配置会恢复为推荐的 5 秒 VFS write-back；自定义缓存配置不变，并且只影响

  真实远端上传进度窗口的可靠性。
- 容量信息增加非交互式远端 `df -Pk` 回退；不会为缺少安全 SSH 路由的密码配置突然弹出密码询问。
- Windows/Linux x64、ARM64 只发布 onefile；macOS x64、ARM64 只发布原生 `.app`，共六个 ZIP。
- alpha.1 非规范包型的用户需要手动安装一次对应的 alpha.2 包。
- macOS ARM64 活动上传期间更新的时序风险仍然保留；本版本不包含实验性的 macOS NFS 后端。
- PR 仍保持 Draft，本预发行不会合并主分支。
