# SSH MountMate v0.4.2

This stable release improves connection organization, mount safety, interactive SSH behavior, and
settings recovery across Windows, macOS, and Linux.

- Connections can be grouped into folders, searched, and sorted. Categorized connections appear
  before uncategorized entries.
- Login auto-mount is now selected per connection. Folder and login preferences can be changed in
  Settings or while a connection is mounted, while connection, authentication, and mount fields
  remain read-only. The home view shows a compact login indicator.
- Interactive SSH and OpenSSH windows stay under application control on Windows. Interactive
  sessions use existing SSH config data, expose retry/end behavior without exiting the app, and are
  excluded from unsupported headless login mounting.
- Custom mountpoints receive stronger local-volume and availability checks. Mount failures remain
  visible with compact details, Retry, and View full log actions instead of being replaced by a
  background status refresh.
- Unreadable settings are preserved in a uniquely named backup and reset to usable defaults. The
  application keeps system-only credential references usable and allows settings to be saved again.
- macOS settings saves no longer touch the login LaunchAgent unless login selections require it.
  Existing custom rclone cache and transfer values remain visible and round-trip without coercion.

---

本稳定版改进了 Windows、macOS 与 Linux 上的连接整理、挂载安全性、交互式 SSH 行为和设置恢复。

- 连接现在可以按文件夹归类、搜索和排序；有分类的连接优先显示在未分类连接之前。
- 登录自动挂载改为逐连接选择。文件夹与登录挂载选项可以在“设置”中修改，也可以在连接已挂载时
  修改；连接、认证和挂载字段仍保持只读。主页只显示紧凑的登录自启标记。
- Windows 上的交互式 SSH 与 OpenSSH 窗口由应用内部管理。交互式会话可以使用已有 SSH config，
  支持重试和结束而不退出主程序，并且不会承诺无法实现的无界面登录自动挂载。
- 自定义挂载点增加了本地卷和可用性检查。挂载失败会持续显示精简原因、“重试”和“查看完整日志”，
  不再被后台状态刷新立即覆盖。
- 无法读取的设置会备份到唯一文件名并重置为可用默认值；仅存在于系统凭据库的引用仍会保持可用，
  用户也可以重新保存设置。
- macOS 普通设置保存不再无条件修改登录 LaunchAgent；已有的自定义 rclone 缓存和传输参数会显示
  原值并按原文保存，不再被错误转换。
