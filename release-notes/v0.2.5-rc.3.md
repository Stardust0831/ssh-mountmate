# SSH MountMate v0.2.5-rc.3

Pre-release focused on the recommended rclone VFS cache profile and connection security.

## Changes

- Change the recommended default mount cache profile to `--vfs-cache-mode full`, enabling local read caching and full write compatibility.
- Pass `--vfs-cache-max-age 30m`, `--vfs-write-back 0s`, and `--dir-cache-time 5m` explicitly by default.
- Keep cache size and minimum free space on their `default (...)` selections, so SSH MountMate does not impose a cache size limit unless the user chooses one.
- Keep the existing fix that skips unreadable `known_hosts_file` paths, avoiding access-denied mount failures on locked-down Windows profiles.
- Pin host keys after their first scan instead of replacing trusted entries on every mount.
- Stop on a host key mismatch instead of retrying with host key validation disabled.
- Write application configuration and rclone credential files atomically with owner-only permissions on POSIX systems.
- Run the unit tests and Python compilation checks before CI build and release packaging.
- Migrate earlier default cache profiles to the new recommended profile while preserving custom cache size and minimum free-space settings.

## Verification

- `PYTHONPATH=src python3 -m unittest discover -s tests -v`
- `git diff --check`

# SSH MountMate v0.2.5-rc.3 中文说明

这个预发布版主要调整 rclone VFS 缓存推荐策略和连接安全性。

## 变化

- 默认推荐挂载缓存策略改为 `--vfs-cache-mode full`，启用本地读缓存，并提供完整写入兼容性。
- 默认显式传递 `--vfs-cache-max-age 30m`、`--vfs-write-back 0s` 和 `--dir-cache-time 5m`。
- 缓存大小和最小剩余空间继续保留 `default (...)` 选项，除非用户主动选择，否则 SSH MountMate 不强行限制缓存大小。
- 保留 `known_hosts_file` 不可读时跳过写入 rclone 配置的修复，避免 Windows 受限用户目录下的拒绝访问导致挂载失败。
- 首次扫描 host key 后固定使用，不再在每次挂载时覆盖已经信任的记录。
- host key 不匹配时停止挂载，不再关闭 host key 校验重试。
- 以原子方式写入应用配置和 rclone 凭据文件，并在 POSIX 系统上限制为仅所有者可读写。
- CI 构建和发布打包前运行单元测试与 Python 编译检查。
- 早期默认缓存策略会迁移到新的推荐策略；用户主动配置过的缓存大小和最小剩余空间会保留。
