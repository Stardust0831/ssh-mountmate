# SSH MountMate v0.3.0-rc.3

Pre-release improving Windows startup status checks and per-connection upload progress.

## Changes

- Replace routine PowerShell CIM process scans with rclone RC PID verification and a Windows native process API fallback.
- Reduce a real 10-configuration, 9-active-mount status check from multi-second worst cases to under 100 ms on the tested Windows system.
- Stop scanning process command lines while resolving an already materialized bundled rclone.
- Show one bottom-right upload progress window per active connection instead of automatically opening the all-mount Transfer center.
- Stack simultaneous connection progress windows and close each one shortly after its upload is confirmed complete.
- Keep the all-mount Transfer center available as a manual view.
- Avoid opening an upload window during the initial two-sample idle confirmation.

## Verification

- Windows native rclone PID enumeration: about 6.3 ms on the tested system.
- Full Windows mount status check: about 88 ms for 10 configurations and 9 active mounts.
- Unit, compile, GUI popup smoke, and packaged executable checks are run before publication.

# SSH MountMate v0.3.0-rc.3 中文说明

这个预发布版优化 Windows 启动挂载检查，并改进按连接显示的上传进度。

## 变化

- 日常状态检查不再调用 PowerShell CIM，改为 rclone RC PID 验证，并使用 Windows 原生进程 API 作为回退。
- 在测试用 Windows 系统上，10 个配置、9 个活动挂载的完整检查从最坏数秒降至 100 毫秒以内。
- 已物化内置 rclone 的常规解析不再扫描全部进程命令行。
- 上传开始时为对应配置显示独立的右下角进度窗，不再自动打开包含全部挂载的传输中心。
- 多个配置同时上传时分别堆叠显示；确认上传完成后，各自的进度窗会延迟关闭。
- 汇总传输中心继续保留为手动查看入口。
- 初始的两次空队列确认不会误弹上传窗口。

## 验证

- Windows 原生 rclone PID 枚举：测试系统约 6.3 毫秒。
- Windows 完整挂载状态检查：10 个配置、9 个活动挂载约 88 毫秒。
- 发布前执行单元测试、源码编译、GUI 浮窗冒烟测试和打包程序检查。
