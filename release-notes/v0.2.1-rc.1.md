# SSH MountMate v0.2.1-rc.1

This prerelease fixes two regressions found after v0.2.0.

## What's Changed

- Fix GUI rclone resolution so bundled rclone is materialized into the managed bin directory before use. GUI mounts should no longer start rclone from PyInstaller `_MEI*` temporary directories.
- Reduce mount-lock and rclone-config-lock timeout failures during parallel mounts by moving slow SSH host-key scanning outside the rclone config file lock.
- Increase rclone config and per-server mount lock wait time to better tolerate parallel mount batches.
- Add a unit test proving bundled rclone resolution returns a managed-bin copy instead of the bundled temporary path.

# SSH MountMate v0.2.1-rc.1 中文说明

这个预发布版修复 v0.2.0 后发现的两个回归问题。

## 主要更新

- 修复 GUI 的 rclone 查找逻辑，确保内置 rclone 会先复制到托管 bin 目录再使用。GUI 挂载不应再从 PyInstaller `_MEI*` 临时目录启动 rclone。
- 将较慢的 SSH host key 扫描移出 rclone config 文件锁，降低并行挂载时 mount lock 和 rclone config lock 超时概率。
- 加长 rclone config 锁和单配置挂载锁等待时间，更适合批量并行挂载。
- 新增单元测试，确认内置 rclone 解析结果是托管 bin 副本，而不是临时 bundled 路径。
