# SSH MountMate v0.4.1

This release improves interactive authentication, update reliability, connection organization,
credential compatibility, and mount diagnostics.

## Interactive SSH authentication

- Passwords, key passphrases, 2FA, host-key confirmation, and other interactive prompts run in an
  application-owned PTY/ConPTY terminal instead of an external console.
- A mount requested during authentication is queued once and resumes automatically when the shared
  SSH session is ready. Repeated clicks do not create duplicate terminals or mounts.
- Terminals can be hidden and reopened without ending the session. Retry, end-session, and
  application-exit behavior use explicit lifecycle controls and warnings.
- Terminal input and output stay in memory and are excluded from application diagnostics.

## Connection organization

- Connections can be assigned to named folders from the connection editor. Existing connections
  remain compatible and appear under Uncategorized until assigned.
- The connection list can be searched by name, folder, SSH alias, host, user, remote path, or
  mountpoint.
- Connections can be sorted stably by saved order, name, or host and remain grouped by folder.
- SSH configuration refresh and overwrite operations preserve user-assigned folders.

## Credentials, diagnostics, and updates

- Passphrase-protected native SFTP keys retain compatible rclone-obscured values while supporting
  the opt-in system credential store. Credential migrations verify before retiring old values.
- Mount failures use compact summaries with retry, dismiss, and full-log actions instead of
  expanding a connection card into a page of diagnostics.
- Prerelease update checks recover from GitHub anonymous REST API rate limits through a bounded
  public Atom feed fallback. Incomplete tags are skipped.
- Automatic installation still requires the production Ed25519 signature, canonical repository
  URLs, matching release version/channel, the full six-platform asset set, exact size, and SHA-256.

## Distribution notes

- Packages include the verified official rclone build.
- Packages are not Authenticode-signed or Apple Developer ID signed/notarized.

本正式版改进了交互式身份验证、更新可靠性、连接整理、凭据兼容性和挂载错误诊断。

## 交互式 SSH 身份验证

- 密码、私钥短语、2FA、主机密钥确认及其他交互提示现在运行在应用自己管理的 PTY/ConPTY
  终端中，不再依赖外部控制台。
- 身份验证期间发起的挂载只排队一次，共享 SSH 会话就绪后会自动继续；重复点击不会创建重复
  终端或挂载。
- 终端可隐藏和重新打开而不结束会话；重试、结束会话及退出应用均有明确的生命周期控制和提示。
- 终端输入输出仅保留在内存中，不进入应用诊断日志。

## 连接整理

- 可以在连接编辑器中把连接归入命名文件夹；旧连接保持兼容，在指定文件夹前显示于“未分类”。
- 可按名称、文件夹、SSH 别名、主机、用户、远端路径或挂载点搜索连接。
- 可按保存顺序、名称或主机稳定排序，并保持文件夹分组显示。
- 刷新或覆盖 SSH 配置时会保留用户指定的文件夹。

## 凭据、诊断与更新

- 原生 SFTP 的加密私钥可继续保留兼容的 rclone obscure 值，并支持可选的系统凭据库；凭据迁移
  会先验证新表示，再清理旧值。
- 挂载失败改为紧凑摘要，并提供重试、关闭和查看完整日志操作，不再让连接卡片被大量错误内容占满。
- GitHub 匿名 REST API 受到限流时，预发行更新检查会使用有大小限制的公开 Atom feed 备用路径，
  并跳过资产尚未完成的 tag。
- 自动安装仍强制验证生产 Ed25519 签名、规范仓库 URL、匹配的版本和发布通道、完整六平台资产、
  精确大小及 SHA-256。

## 分发说明

- 安装包包含经过校验的官方 rclone 构建。
- 安装包尚未使用 Authenticode 或 Apple Developer ID 签名/公证。
