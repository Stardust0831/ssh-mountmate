# SSH MountMate v0.3.0

First stable v0.3 release with truthful upload visibility, fast Windows status checks, Explorer refresh integration, dual packaging, and verified self-updates.

## Highlights

- Show each mount's real rclone VFS queue, remote bytes transferred, speed, and active-file progress.
- Show one non-activating bottom-right progress window per uploading connection; simultaneous uploads stack independently.
- Warn before unsafe unmount or exit operations while uploads remain queued or their state cannot be confirmed.
- Verify remote refreshes by forgetting cached entries, refreshing VFS, and directly listing the remote directory.
- Add connection-card and Windows Explorer context-menu actions for refresh, transfers, open, and logs.
- Fix Explorer refresh at Windows drive roots by safely normalizing trailing separators and quotes.
- Replace routine PowerShell CIM mount checks with parallel rclone RC verification and a Windows native process API fallback.
- Publish both backward-compatible onefile packages and faster-starting `-onedir.zip` packages for all six targets.

## Verified self-update

- Automatically select the same onefile or onedir package type for the current OS and CPU architecture.
- Download only HTTPS GitHub assets and verify GitHub's published SHA-256 digest and file size.
- Reject ZIP path traversal, absolute paths, drive paths, symbolic links, oversized archives, and excessive file counts.
- Stage the update beside the current installation before asking to restart.
- Replace the application only after the current process exits, verify that the new process remains alive, and roll back automatically on startup failure.
- Keep existing rclone mounts and uploads running while the GUI restarts.
- Fall back to the release page when the build is unpackaged, launched from a ZIP temporary directory, installed in a non-writable location, or missing a trusted digest.

## Review and verification

- All 85 unit tests plus compile, Ruff, whitespace, GUI, package, live GitHub download, SHA-256, extraction, staging, and updater-script checks.
- Windows, Linux, and macOS builds for x64 and arm64.

# SSH MountMate v0.3.0 中文说明

首个 v0.3 正式版，新增真实上传可见性、快速 Windows 状态检查、资源管理器刷新、双包型构建和可信自更新。

## 主要变化

- 显示每个挂载真实的 rclone VFS 队列、远端已传字节、速度和活动文件进度。
- 每个正在上传的配置显示独立且不抢焦点的右下角进度窗；多个上传分别堆叠。
- 上传仍在排队或状态无法确认时，取消挂载或退出会先警告。
- 刷新会清除缓存、刷新 VFS，并直接列出远端目录进行核验。
- 连接卡片和 Windows 资源管理器右键菜单支持刷新、传输、打开和日志操作。
- 修复 Windows 盘符根目录刷新时末尾分隔符和引号解析错误。
- 日常挂载检查不再使用 PowerShell CIM，改为并行 rclone RC 验证和 Windows 原生进程 API 回退。
- 六个平台目标同时发布兼容原命名的 onefile 包和启动更快的 `-onedir.zip` 包。

## 可信自更新

- 根据当前系统、CPU 架构和 onefile/onedir 包型自动选择相同类型的更新附件。
- 只下载 GitHub HTTPS 附件，并校验 GitHub 发布的 SHA-256 摘要和文件大小。
- 拒绝 ZIP 路径穿越、绝对路径、盘符路径、符号链接、超大解压体积和异常文件数量。
- 用户确认重启前，先把更新完整暂存在当前安装目录旁。
- 当前进程退出后才替换程序；新进程启动失败时自动恢复并重新启动旧版本。
- GUI 重启期间，已有 rclone 挂载和上传继续运行。
- 未打包构建、从 ZIP 临时目录直接启动、安装目录不可写或附件缺少可信摘要时，回退到 Release 页面手动更新。

## 审查与验证

- 执行完整单元测试、源码编译、Ruff、空白检查、GUI、打包、GitHub 实际下载、SHA-256、解压、暂存和更新脚本验证。
- Windows、Linux、macOS 的 x64 和 arm64 构建。
