# SSH MountMate v0.2.5-rc.2

Pre-release focused on directory-cache refresh and restoring SSH MountMate's original cache defaults.

## Changes

- Make the main Refresh button refresh rclone directory/metadata caches for currently mounted configs by calling `vfs/forget`, then refresh GUI status and capacity.
- Keep directory-cache refresh failures non-blocking; failed refresh details are written to `refresh.log`.
- Restore SSH MountMate's original VFS cache default: mounts explicitly pass `--vfs-cache-mode writes`.
- Restore the original `default (...)` selections for cache size, minimum free space, write-back delay, and directory cache time, so SSH MountMate no longer shortens or limits them unless the user chooses a value.
- Only pass `--vfs-write-back` when VFS cache mode is `writes` or `full`; the setting is disabled in the UI for `off` and `minimal`.
- Skip unreadable `known_hosts_file` paths instead of writing them into rclone config, avoiding access-denied mount failures on locked-down Windows profiles.
- Migrate earlier RC cache defaults back to the original `writes` default while preserving custom cache settings.

# SSH MountMate v0.2.5-rc.2 中文说明

这个预发布版主要调整目录缓存刷新，并恢复 SSH MountMate 旧版缓存默认值。

## 变化

- 主界面的“刷新”现在会对已挂载配置调用 `vfs/forget`，刷新 rclone 的目录/元数据缓存，然后再刷新 GUI 状态和容量。
- 目录缓存刷新失败不会阻断整体刷新；失败详情会写入 `refresh.log`。
- 恢复 SSH MountMate 旧版 VFS 缓存默认值：挂载时明确传递 `--vfs-cache-mode writes`。
- 缓存大小、最小剩余空间、写回延迟和目录缓存时间恢复为旧版的 `default (...)` 选项，除非用户主动选择，否则 SSH MountMate 不再额外缩短或限制这些参数。
- 只有在 VFS 缓存模式为 `writes` 或 `full` 时才传递 `--vfs-write-back`；`off` 和 `minimal` 下 UI 会禁用该设置。
- `known_hosts_file` 路径不可读时不再写入 rclone 配置，避免 Windows 受限用户目录下的拒绝访问导致挂载失败。
- 早期 RC 的缓存默认值会迁回旧版 `writes` 默认值；用户主动配置过的缓存设置会保留。
