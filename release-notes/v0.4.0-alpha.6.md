# SSH MountMate v0.4.0-alpha.6

This prerelease fixes Windows self-update startup failure `os error 740` reported when
`v0.4.0-alpha.4` attempted to update.

## Windows update helper

- Embeds an explicit `requestedExecutionLevel level="asInvoker"` manifest in official Windows
  executables, preventing Windows from heuristically treating the application as an elevated
  installer.
- Renames the detached self-update executable from `SSHMountMate-updater-*` to the neutral
  `SSHMountMate-helper-*` form so Windows Installer Detection is not triggered by its filename.
- Windows CI now extracts the manifest from the real PE executable and rejects missing manifests,
  `requireAdministrator`, and `highestAvailable` execution levels.
- Existing update commit, rollback, health confirmation, and active-mount preservation tests remain
  required on Windows x64 and ARM64.

## One-time manual update required

`v0.4.0-alpha.4` and `v0.4.0-alpha.5` create the old helper before any downloaded application code
can run. They therefore cannot repair this failure through their own automatic updater. Install this
release manually once; subsequent prerelease self-updates will use the corrected helper.

## Important prerelease limitations

- Packages are not yet production code-signed or notarized.
- This release does not include an installer or the planned Experimental macOS NFS backend.
- The Rust rewrite PR remains Draft and is not merged by this release.

本预发行版修复 `v0.4.0-alpha.4` 更新时出现的 Windows 自更新启动错误 `os error 740`。

- 官方 Windows EXE 现在嵌入明确的 `requestedExecutionLevel level="asInvoker"` manifest，避免
  Windows 将程序启发式识别为需要管理员权限的安装器。
- 分离式更新 helper 从 `SSHMountMate-updater-*` 改名为中性的 `SSHMountMate-helper-*`，避免
  文件名触发 Windows Installer Detection。
- Windows CI 会从真实 PE 文件中抽取 manifest，并拒绝缺失 manifest、`requireAdministrator`
  或 `highestAvailable` 权限级别。
- `alpha.4` 和 `alpha.5` 会在新版本代码运行前先启动旧 helper，因此无法通过自身的自动更新
  修复该问题。需要手动安装本版本一次；之后的预览版自更新会使用修复后的 helper。
- 本版本不包含安装器或实验性 macOS NFS 后端，PR 仍保持 Draft。
