# SSH MountMate v0.1.7-rc.1

This pre-release is for testing custom mountpoints and duplicate detection changes.

## What's Changed

- Add an explicit custom mountpoint option in the Add/Edit config dialog.
- Allow choosing a parent folder for custom mountpoints; SSH MountMate fills a generated child mountpoint such as `parent/name`, with suffixes when needed.
- Keep Auto, `~/mnt/name`, drive-letter presets, and custom folder mountpoints in the same mountpoint control.
- Allow new SSH config batch import entries to be manually ignored instead of imported.
- Relax duplicate detection:
  - configs are only blocked as duplicates when IP/Host, user, port, remote path, and mountpoint are all the same;
  - the same SSH server can now be saved multiple times for different remote paths or local mountpoints.
- Keep SSH config batch import conservative: an imported Host is treated as duplicate when it points to the same HostName/User/Port, because batch import does not ask for per-entry remote paths or local mountpoints.

## Notes

- On Windows, custom folder mountpoints still require an existing parent directory and a target folder that does not already exist.
- On macOS/Linux, custom folders are created if missing, but they must be valid absolute paths or `~` paths.

# SSH MountMate v0.1.7-rc.1 中文说明

这个预发布版用于测试自定义挂载点和重复配置判断调整。

## 主要更新

- 新增/编辑配置页增加明确的自定义挂载点选项。
- 自定义挂载点支持选择父文件夹；SSH MountMate 会自动填入类似 `父文件夹/名称` 的子挂载点，必要时自动加后缀。
- Auto、`~/mnt/名称`、Windows 盘符预设和自定义文件夹现在都在同一个挂载点控件里。
- SSH config 批量导入中的新条目现在也可以手动选择忽略，而不是只能导入。
- 放宽重复配置判断：
  - 只有 IP/主机、用户名、端口、远程路径和挂载点都相同时，才会拦截为重复配置；
  - 同一台 SSH 服务器现在可以为不同远程路径或不同本地挂载点保存多份配置。
- SSH config 批量导入保持保守判断：只要导入的 Host 指向相同 HostName/User/Port，就认为是重复，因为批量导入不会逐项填写远程路径或本地挂载点。

## 注意

- Windows 自定义文件夹挂载点仍要求父目录已存在，目标文件夹本身不能已存在。
- macOS/Linux 自定义文件夹不存在时会创建，但必须是合法的绝对路径或 `~` 路径。
