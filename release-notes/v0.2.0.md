# SSH MountMate v0.2.0

This release keeps SSH MountMate as a single-executable app while improving rclone process handling, Windows packaging behavior, and narrow-card UI layout.

## What's Changed

- Keep PyInstaller `onefile` release packaging.
- Materialize the bundled rclone binary into a stable managed bin directory before starting mounts, avoiding PyInstaller `_MEI*` temporary-directory cleanup warnings while mounts remain active.
- Use hash-named managed rclone copies so active mounts are not overwritten during upgrades.
- Clean up old managed rclone copies on Windows, macOS, and Linux while preserving copies still referenced by running rclone processes.
- Store managed rclone copies under:
  - Windows: `%LOCALAPPDATA%\SSHMountMate\bin`
  - macOS: `~/Library/Application Support/SSHMountMate/bin`
  - Linux: `${XDG_DATA_HOME:-~/.local/share}/ssh-mountmate/bin`
- Close parent GUI/CLI log-file handles immediately after starting `rclone mount`.
- Use rclone process command-line matching on Windows, macOS, and Linux instead of relying only on saved PIDs.
- Replace the old Windows CLI `sleep(2)` mount check with readiness polling shared with the GUI mount path.
- Fix narrow connection-card action-button layout while keeping the four buttons pinned to the right side.
- Add focused unit tests for rclone command matching, procfs process discovery, mount status decisions, and mount readiness polling.

## Notes

- Thanks to @bane-dysta for reporting the Windows `_MEI*` cleanup warning and proposing the packaging workaround.

# SSH MountMate v0.2.0 中文说明

这个版本在保持单文件可执行程序形态的同时，改进了 rclone 进程处理、Windows 打包运行行为和较窄窗口下的卡片按钮布局。

## 主要更新

- Release 构建继续使用 PyInstaller `onefile` 单文件模式。
- 启动挂载前，会先把内置 rclone 复制到稳定的托管 bin 目录，避免活跃挂载导致 PyInstaller `_MEI*` 临时目录无法清理。
- 托管 rclone 使用带哈希的文件名，升级时不会覆盖正在被活跃挂载使用的旧 rclone。
- Windows、macOS 和 Linux 都会清理旧的托管 rclone 副本，同时保留仍被运行中 rclone 进程引用的副本。
- 托管 rclone 保存位置：
  - Windows：`%LOCALAPPDATA%\SSHMountMate\bin`
  - macOS：`~/Library/Application Support/SSHMountMate/bin`
  - Linux：`${XDG_DATA_HOME:-~/.local/share}/ssh-mountmate/bin`
- 启动 `rclone mount` 后立即关闭 GUI/CLI 父进程持有的日志文件句柄。
- Windows、macOS 和 Linux 都会读取 rclone 进程命令行，不再只依赖保存的 PID 判断挂载状态。
- 旧 Windows CLI mount 路径不再使用固定 `sleep(2)`，改为和 GUI 共用 readiness 轮询。
- 修复较窄连接卡片里的操作按钮布局，同时保持四个按钮固定在卡片最右侧。
- 新增针对性单元测试，覆盖 rclone 命令匹配、procfs 进程发现、挂载状态判断和 readiness 轮询。

## 注意

- 感谢 @bane-dysta 报告 Windows `_MEI*` 清理警告，并提出打包方式调整方案。
